# Trainer Design Document v1.0

## 0. 目的

mol Trainerを基準モデルとして、今後の電磁気・熱・力学などの教材を、同じ構造・操作性・ログ形式で追加できる共通仕様を定める。

---

# 1. 教材共通フロー

## 1-1. 学生が進む順番

1. **Study Notes**
   - 単元の概念を短時間で理解する。
   - Pointごとにページを分ける。
   - 各Pointには必要に応じてTipsを含める。

2. **Summary**
   - Study Notesで扱った内容を一覧で確認する。
   - 新しい説明は追加せず、重要事項だけを短く整理する。

3. **Check Quiz**
   - 3問程度。
   - Study Notesを読めば答えられる基本確認。
   - 演習に入る前の最低限の理解を確認する。

4. **はじめる前に**
   - 単位変換、指数、電卓操作など、その単元の前提技能を確認する。
   - mol Trainerでは mL⇔L、mg⇔g、mol/Lの意味などを扱う。

5. **基礎①〜③**
   - 1テーマ5問。
   - 単一技能を反復する。
   - 4問以上正解でクリア。

6. **実践①〜③**
   - 実験・生命科学・複数段階計算につながる問題。
   - 必要に応じて関連する複数設問を1ページに表示する。
   - 設問ごとに回答・解説を確認しながら進む。

7. **Complete**
   - 単元全体の到達状況を表示する。
   - 未クリア項目と再挑戦先を示す。

## 1-2. mol Trainerでの対応例

| 区分 | 内容 |
|---|---|
| Study Notes | mol、モル質量、質量とmolの関係 |
| Summary | 粒子数・物質量・質量の関係、生命科学での利用 |
| Check Quiz | 概念確認3問 |
| はじめる前に | 単位変換 |
| 基礎① | molと粒子数 |
| 基礎② | モル質量 |
| 基礎③ | 質量⇔mol |
| 実践① | 溶液調製 |
| 実践② | 希釈 |
| 実践③ | 総合・実験問題 |

# 2. Study Notesテンプレート

## 2-1. 基本構造

- Pointごとに1ページ。
- 1ページにつき中心概念は1つ。
- タイトル下に十分な余白を取る。
- 説明文は原則として段落で書き、不要な改行を入れない。
- Tipsは独立ページにせず、関連するPoint内に置く。
- イラストは理解を助ける場合だけ使用し、文章量を多く見せない。

## 2-2. Pointの型

1. Point番号とタイトル
2. 導入説明
3. 必要最小限の図・例
4. Tips
5. 「次へ」ボタン

## 2-3. Summaryの型

- Study Notesとは別ページ。
- 重要事項を短いリストで整理する。
- 図は概念関係を示す簡潔なものだけにする。
- 「生命科学での利用」などの動機づけは、Summaryと同一ページ内の別カードにする。

## 2-4. Check Quizの型

- 3問。
- 4択を基本とする。
- 回答直後に正誤と短い解説を表示する。
- 全問終了後に「はじめる前に」または基礎①へ進む。

# 3. 演習問題テンプレート

## 3-1. 基礎問題

- 1画面1問。
- 問題文は他画面と同じ本文サイズ。
- 選択肢を選び、「答える」を押す。
- 回答後に以下を表示する。
  - 正誤
  - 正解
  - 1〜2文の解説
  - 「次の問題へ」
- 5問で1セット。
- 4問以上正解でクリア。

## 3-2. 実践問題

### 実践問題の共通構成

- 実践問題では、最終結果だけを一度に求めさせず、解き方を誘導する複数の設問に分ける。
- 各設問は、単位変換、物質量の算出、質量または体積の算出など、1つの思考段階に対応させる。
- 関連する設問は同一ページに表示し、設問ごとに「答える」ボタンを設置する。
- 回答後は、その設問の直下に正誤と計算過程を表示する。
- 全設問の終了後に、計算結果を実際の実験操作へ結びつけた「まとめ」を表示する。
  - 溶液調製の例：「溶質X gを量り取り、水に溶かして全体をY mLにすると、Z mol/Lの溶液が完成します。」
  - 希釈の例：「元液X mLに水Y mLを加えると、Z mol/Lの溶液が完成します。」


- 1つの状況に関連する複数設問を、同一ページに表示してよい。
- 各設問に個別の「答える」ボタンを付ける。
- 回答後、その設問の直下に解説を表示する。
- 前の設問の考え方を確認しながら、次の設問へ進める。
- 全設問終了後に、実験操作としての最終結果を文章で表示する。

### mol Trainer 希釈問題の例

1. 目標濃度1 Lに含まれる物質量
2. 実際に作る体積に含まれる物質量
3. 元液の必要体積
4. 最終表示：元液X mLに水Y mLを加える

## 3-3. 問題文ルール

- 必要な値は問題文内に明記する。
- 例：
  - 「CO₂のモル質量は44 g/molです。CO₂ 22 gは何molですか。」
  - 「NaClのモル質量は58.5 g/molです。NaCl 2 molの質量は何gですか。」
- 初学者に不要な情報は入れない。
- 単位は必ず記載する。
- 解説では、計算式だけでなく「何を求めているか」を文章で補う。

# 4. デザインルール

## 4-1. 色の役割

| 用途 | 色の役割 |
|---|---|
| Study Notes | 青系 |
| はじめる前に | 緑系 |
| 基礎 | 青系 |
| 実践 | オレンジ系 |
| 正解 | 緑 |
| 不正解 | 赤 |
| 注意・Tips | 淡い青または黄 |

## 4-2. カード

- 背景は白。
- 枠線は薄いグレー。
- 角丸と影は全ページで統一する。
- Pointごとにカードデザインを変えない。
- カード内余白はPC・スマホで窮屈にならない値にする。

## 4-3. 文字

- Pointタイトルは全Pointで同じサイズ。
- タイトルと本文の間に1行分程度の余白。
- 本文は読みやすい行間を確保する。
- 説明文は太字を多用しない。
- Summary本文は基本的に細字。重要語だけ必要最小限で強調する。

## 4-4. 操作

- 主操作ボタンは画面下部またはカード末尾に配置。
- ボタン文言を統一する。
  - 「次へ」
  - 「答える」
  - 「次の問題へ」
  - 「もう一度挑戦」
- スマホで押しやすい高さを確保する。

## 4-5. トップページ

学習順が分かるように、以下の区分で表示する。

- Study Notes
- Check Quiz
- はじめる前に
- 基礎①〜③
- 実践①〜③

各カードには以下を表示する。

- 区分名
- 学習内容
- 短いゴール
- クリア状況
- 最高得点または進捗

# 5. ログ設計

## 5-1. 現在のGASで取得できている情報

添付GASでは、大学Googleアカウントから認証トークンを発行し、メールアドレスそのものではなく学籍番号を記録する。

現在の主な記録項目：

- 学籍番号
- Stage名
- Event名
- 得点
- 問題数
- 正答率
- 所要時間
- 日時
- クリア判定
- ページ名
- User-Agent
- 画面サイズ
- Payload全体
- Stage別挑戦回数
- 全体挑戦回数
- 最終更新日時

現在の主なEvent：

- `study_check_complete`
- `stage_result`

GASには以下の表示機能もある。

- 学生本人用ダッシュボード
- 教員用ダッシュボード
- Stage別集計
- 挑戦回数集計

## 5-2. 共通ログライブラリで取得可能な情報

添付の `trainer_common_v2.js` では、以下を記録できる。

- セッション全体の所要時間
- 各設問の回答時間
- 各設問の正誤
- 選択した回答
- 正解
- 問題文
- 問題ID
- 問題タイプ
- User-Agent
- 画面サイズ

さらに、タブが非表示になっている時間を所要時間から除外できる。

## 5-3. v1.0で統一するEvent

| Event | 発生タイミング |
|---|---|
| `study_open` | Study Notes開始 |
| `study_point_complete` | 各Point読了 |
| `study_summary_complete` | Summary読了 |
| `study_check_complete` | Check Quiz完了 |
| `module_start` | はじめる前に・基礎・実践開始 |
| `question_answer` | 各設問回答 |
| `module_complete` | 5問または実践問題完了 |
| `retry_start` | 再挑戦開始 |
| `trainer_complete` | 単元全体完了 |

## 5-4. v1.0の共通Payload

```javascript
{
  trainerId: "mol",
  trainerVersion: "1.0.0",
  mode: "self",
  moduleId: "basic_01",
  moduleGroup: "basic",
  moduleTitle: "molと粒子数",
  attemptId: "...",
  score: 4,
  total: 5,
  passed: true,
  elapsed: 185,
  firstAnswerDelay: 22,
  questionTimes: [12, 25, 18, 41, 20],
  questionResults: [...],
  completedAt: "..."
}
```

## 5-5. 特に追加したい指標

### 初動時間

ページを開いてから最初の回答までの時間。

- 初動が遅い学生
- 画面は開いたが取り組んでいない学生

を把握するために使用する。

### 最終操作時刻

授業モードのリアルタイム進捗表示で使用する。

### Point読了状況

Study Notesを開いただけでなく、どこまで進んだかを記録する。

### 設問別正答率

問題IDを固定し、問題ごとの正答率・平均回答時間を集計する。

### 再挑戦

初回と再挑戦を区別し、学習による改善を見る。

## 5-6. 現状からの重要な修正点

現在のmol Trainerでは、Stage全体の経過時間は `Date.now()` で計測しているため、タブを離れている時間も含まれる。

一方、`trainer_common_v2.js` には非表示時間を除外する仕組みがある。v1.0ではmol Trainer側の独自計測をやめ、共通ログライブラリへ統一する。

また、現在のmol TrainerはStage終了時の結果送信が中心で、設問ごとの時間・回答内容はGASへ送っていない。v1.0では `questionResults` と `questionTimes` を共通Payloadとして送信する。

# 6. 実装方針

## Phase 1：mol Trainer v1.0

- 名称を「はじめる前に・基礎①〜③・実践①〜③」に統一
- Study Notesテンプレート確定
- 問題画面テンプレート確定
- デザインルール確定
- `MolTracker` と `TrainerLog` を統合
- 共通Event・Payloadに変更
- GASのシート列とダッシュボードを新仕様へ対応

## Phase 2：Trainer Engine化

- 教材内容を設定ファイルへ分離
- 共通HTML・CSS・JSを固定
- 新教材は以下だけで追加できる構造にする
  - Study Notesデータ
  - Summaryデータ
  - Check Quizデータ
  - 問題バンク
  - 教材タイトル・色・アイコン

## Phase 3：電磁気版で検証

- mol Trainerと同じ構造で電磁気教材を1単元作成
- 操作性・ログ・授業内利用を確認
- 必要な修正をTrainer Engineへ戻す

---

# 7. v1.0完成条件

- 学生が迷わず学習順をたどれる。
- PC・スマホで同じ機能が使える。
- Study Notes、基礎、実践のUIが統一されている。
- 教材ごとにHTMLを作り直さず、データ差し替えで追加できる。
- 大学アカウントで学籍番号を安全に取得できる。
- 各設問の正誤・回答時間・Stage所要時間が記録される。
- タブ非表示時間を学習時間に含めない。
- 初動時間、再挑戦、進捗を教員が確認できる。
- 学生本人が自分の進捗を確認できる。
