# タグ設計と運用メモ

My Lab の興味検索は、研究室ごとのタグ品質に大きく左右されます。新しい学科や研究室を追加するときは、研究室名の印象ではなく、公式研究室紹介ページの本文を基準にしてタグを付けます。

タグは単語抽出ではなく、検索用メタデータを編集する作業です。高校生・在校生が、研究対象・学問領域・研究手法から横断的に研究室を探せることを優先します。

## 基本方針

1. 公式研究室紹介ページの本文を読む。
2. 研究対象から `major_categories` を決める。
3. `tags.targets`、`tags.fields`、`tags.methods` に分けてタグを作る。
4. `keywords` は `tags.targets` → `tags.fields` → `tags.methods` の順に連結する。
5. 固有名詞だけでなく、他研究室とも比較できる語を残す。
6. 検索で拾えない専門語は、`app.js` の `tagParents` に上位タグを追加する。

## 情報源の優先順位

1. 東洋大学公式研究室紹介ページ
2. 教員本人の researchmap
3. KAKEN研究者情報・採択課題
4. 研究室ホームページ
5. 代表論文・代表プロジェクト

研究室名だけからタグを推測しません。

## `major_categories`

大区分は、ユーザーが最初に研究室を大きく探すための分類です。

優先して使う分類:

- `動物`
- `植物`
- `微生物`

必要に応じて追加する分類:

- `細胞`
- `環境`
- `健康`
- `化学`

研究室名ではなく、実際の研究対象で決めます。たとえば「地球環境科学研究室」でも、本文がイネ科植物の鉄吸収機構を中心にしている場合は `植物` を付けます。

## `tags`

`tags` は、研究対象・学問領域・研究手法に分けます。

### `tags.targets`

研究対象を3〜4個程度入れます。

例: `動物`, `植物`, `微生物`, `DNA`, `遺伝子`, `細胞`, `神経`, `骨格`, `筋肉`, `鳥類`, `哺乳類`, `イネ`, `水環境`

### `tags.fields`

学問領域を2〜4個程度入れます。

例: `比較解剖学`, `形態学`, `生態学`, `生理学`, `神経科学`, `発生生物学`, `分子生物学`, `遺伝学`, `微生物学`, `環境科学`, `毒性学`, `バイオメカニクス`

### `tags.methods`

研究手法を2〜3個程度入れます。

例: `フィールド調査`, `行動観察`, `解剖`, `細胞培養`, `分子実験`, `遺伝子解析`, `顕微鏡`, `CT`, `統計解析`, `データ解析`, `化学分析`, `ゲノム解析`

装置名が研究室の特徴を表す場合は、手法タグとして使ってよいです。

## `keywords`

`keywords` は `tags.targets`、`tags.fields`、`tags.methods` をこの順番で連結した表示・検索用配列です。手入力で別管理せず、`tags` から再生成します。

避けたい語:

- `生命科学`
- `生物`
- `研究`
- `医療`
- `健康`
- `環境`
- `実験`
- `データ`
- 文章中に一度だけ出る周辺語
- 研究内容の中心ではない一般語
- その研究室だけで閉じていて、検索入口になりにくい細かすぎる語

広すぎる語は、より具体的な語に置き換えます。たとえば `環境` ではなく `水環境`、`環境科学`、`環境分析`、`環境保健学` のようにします。

ただし、重要な固有名詞は残してよいです。その場合は上位語や関連する領域・手法も一緒に入れます。

例:

- `アカパンカビ` だけでなく、`糸状菌`, `DNA修復`, `分子遺伝学`, `遺伝子解析` も入れる。
- `ムギネ酸類` だけでなく、`イネ`, `植物栄養学`, `鉄`, `顕微鏡` も入れる。
- `発電菌` だけでなく、`電気活性微生物`, `微生物生理学`, `電気化学測定` も入れる。

## 上位タグ補完

専門語を選んだときに大きなトピックへつながるよう、`app.js` の `tagParents` で補完します。

例:

- `イネ科植物` -> `イネ`, `植物`, `作物`
- `発電菌` -> `電気活性微生物`, `電気微生物`, `微生物`
- `糖鎖間相互作用` -> `糖鎖`, `細胞接着`, `細胞`

新しい学科を追加したあと、期待したトピックに研究室が出ない場合は、まず `major_categories` と `tags` を確認します。それでも弱い場合だけ `tagParents` を増やします。

## 点検チェック

学科追加後は、最低限以下を確認します。

- すべての研究室に `major_categories` がある。
- すべての研究室に `tags.targets`、`tags.fields`、`tags.methods` がある。
- `keywords` が `tags.targets` → `tags.fields` → `tags.methods` の順になっている。
- `keywords` に避けたい広すぎる語が単独で入っていない。
- `動物`, `植物`, `微生物` を選んだとき、明らかに違う研究室だけが上位に出ていない。
- 専門語を選んだとき、上位カテゴリの研究室も自然に拾える。
- JSON構文とJavaScript構文が壊れていない。
