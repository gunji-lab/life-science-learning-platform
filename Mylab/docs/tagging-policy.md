# タグ設計と運用メモ

My Lab の興味検索は、研究室ごとのタグ品質に大きく左右されます。新しい学科や研究室を追加するときは、研究室名の印象ではなく、公式研究室紹介ページの本文を基準にしてタグを付けます。

## 基本方針

1. 公式研究室紹介ページの本文を読む。
2. 研究対象から `major_categories` を決める。
3. 研究内容から `keywords` を5〜10個に絞る。
4. 固有名詞だけでなく、他研究室とも比較できる語を残す。
5. 検索で拾えない専門語は、`app.js` の `tagParents` に上位タグを追加する。

## `major_categories`

大区分は、ユーザーが最初に研究室を大きく探すための分類です。

優先して使う分類:

- `動物`
- `植物`
- `微生物`

必要に応じて追加する分類:

- `細胞`
- `環境`
- `健康`
- `化学`

研究室名ではなく、実際の研究対象で決めます。たとえば「地球環境科学研究室」でも、本文がイネ科植物の鉄吸収機構を中心にしている場合は `植物` を付けます。

## `keywords`

`keywords` は、研究内容を表す5〜10個程度の語にします。無理に多くしすぎると、関係の薄い研究室まで検索に出やすくなります。

優先する語:

- 研究対象: `イネ`, `海鳥`, `糸状菌`, `iPS細胞`
- 研究テーマ: `DNA修復`, `環境浄化`, `植物ホルモン`, `糖鎖間相互作用`
- 方法や視点: `フィールド調査`, `顕微鏡観察`, `統計モデリング`, `電気化学`
- 高校生にも入口になりやすい共通語: `脳`, `心臓`, `食料`, `健康`, `生態系`

避けたい語:

- 文章中に一度だけ出る周辺語
- 研究内容の中心ではない一般語
- その研究室だけで閉じていて、検索入口になりにくい細かすぎる語

ただし、重要な固有名詞は残してよいです。その場合は必ず上位語も一緒に入れます。

例:

- `アカパンカビ` だけでなく、`糸状菌`, `微生物`, `DNA修復` も検索に効くようにする。
- `ムギネ酸類` だけでなく、`イネ`, `植物栄養`, `鉄吸収` も入れる。
- `発電菌` だけでなく、`電気活性微生物`, `微生物`, `電子伝達` も入れる。

## 上位タグ補完

専門語を選んだときに大きなトピックへつながるよう、`app.js` の `tagParents` で補完します。

例:

- `イネ科植物` -> `イネ`, `植物`, `作物`
- `発電菌` -> `電気活性微生物`, `電気微生物`, `微生物`
- `糖鎖間相互作用` -> `糖鎖`, `細胞接着`, `細胞`

新しい学科を追加したあと、期待したトピックに研究室が出ない場合は、まず `major_categories` と `keywords` を確認します。それでも弱い場合だけ `tagParents` を増やします。

## 点検チェック

学科追加後は、最低限以下を確認します。

- すべての研究室に `major_categories` がある。
- `keywords` が5〜10個に収まっている。
- `動物`, `植物`, `微生物` を選んだとき、明らかに違う研究室だけが上位に出ていない。
- 専門語を選んだとき、上位カテゴリの研究室も自然に拾える。
- JSON構文とJavaScript構文が壊れていない。
